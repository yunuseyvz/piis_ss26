# FlowQuest JupyterLab extension

The frontend + server-extension package that turns a notebook into a quest.

## Structure

```
jupyterlab_extensions/piis-assistant/
├── jupyterlab_piis_assistant/         Python server extension
│   ├── analyzer.py                    AST-based notebook analysis (regions, issues, missions, activity slots)
│   ├── ai_backend.py                  LLM client + prompt flows (explain, reflect, next-steps, quiz JSON helpers)
│   ├── activities.py                  Between-cell activity registry + generation/grading (quiz/predict/teachback)
│   ├── gamification.py                Pure XP/level state mutations (award XP, normalise, public view)
│   ├── progress_store.py              Global progression persistence (~/.flowquest/progress.json)
│   ├── handlers.py                    Tornado routes registered at /piis-assistant/...
│   ├── settings.py                    Global model / API key persisted in ~/.flowquest/settings.json
│   └── labextension/                  Built static assets (generated by `npm run build`)
├── src/                               TypeScript frontend
│   ├── index.ts                       JupyterFrontEnd plugin entrypoint; owns the global state mirror
│   ├── api.ts                         Tiny request helper + escape/award-key utilities
│   ├── types.ts                       Shared interfaces (QuestState mirrors gamification.public_view)
│   ├── questState.ts                  EMPTY_QUEST_STATE constant
│   ├── questStore.ts                  metadata.flowquest read/write (difficulty + quizzes only)
│   ├── notebookContext.ts             NotebookPanel → analyser payload + chat context
│   ├── notebookBanner.ts              In-notebook HUD banner
│   ├── cellDecorations.ts             Per-cell chip + inline expandable panel
│   ├── questCells.ts                  Virtual activity cells injected between real cells
│   ├── cellModules/                   Per-activity renderers (choice / open) + registry
│   ├── avatarAssistant.ts             Flowy — floating, paste-aware companion
│   ├── flowySprite.ts                 SVG sprite for Flowy's moods
│   ├── markdown.ts                    Escape-first Markdown → HTML for chat replies
│   ├── uiFeedback.ts                  Shared error / spinner / thinking markup
│   ├── settingsPanel.ts               Settings modal (global endpoint + per-notebook difficulty)
│   └── sidebar.ts                     Quest / Flowy / Chat sidebar tabs
├── style/index.css                    Sharp-edged design language using JupyterLab theme vars
├── package.json                       Frontend build, JupyterLab schema
├── setup.py                           Packages the labextension/ assets into a wheel
└── tsconfig.json                      TypeScript options
```

## Build + install for local development

The easiest path is the bootstrap script at the repo root:

```bash
uv sync
bash scripts/setup.sh
uv run jupyter lab examples
```

Re-run `bash scripts/setup.sh` after any `uv sync` or extension code change.

If you prefer to run the steps by hand:

```bash
uv sync
cd jupyterlab_extensions/piis-assistant
npm install
PATH="$(pwd)/../../.venv/bin:$PATH" npm run build
uv pip install --python ../../.venv/bin/python .
cd ../..
uv run jupyter lab examples
```

Verify the extension is installed:

```bash
uv run jupyter labextension list | grep piis
# jupyterlab-piis-assistant v0.1.0 enabled OK (python, jupyterlab-piis-assistant)
```

Iterating:

```bash
# terminal 1: continuous TS compile
cd jupyterlab_extensions/piis-assistant
npm run watch

# terminal 2: rebuild and reinstall after backend or asset changes
PATH="../../.venv/bin:$PATH" npm run build
uv pip install --python ../../.venv/bin/python .
```

Refresh the JupyterLab tab to pick up frontend changes; restart the Jupyter server for backend changes.

## Server routes

All routes live under `/piis-assistant/` and use the standard Jupyter authentication.

| Route | Method | Purpose |
| --- | --- | --- |
| `/status` | GET | Endpoint configuration check. |
| `/chat` | POST | Notebook-aware chat. |
| `/analyze` | POST | Run the AST analysis, apply auto-checks, return the global state. |
| `/quest/init` | GET/POST | View the current global progression. |
| `/mission/claim` (alias `/quest/claim`) | POST | Idempotent XP award for a mission (per notebook). |
| `/explain-cell` | POST | LLM cell explanation, awards exploration XP (first time per cell). |
| `/reflect/prompt` | POST | LLM produces a reflective question. |
| `/reflect/answer` | POST | Persist the reflection, award XP. |
| `/next-steps` | POST | LLM suggests three next steps. |
| `/quiz/generate` (alias `/activity/generate`) | POST | Generate a between-cell activity (quiz/predict/teachback). |
| `/quiz/answer` (alias `/activity/answer`) | POST | Record an attempt, award XP. |
| `/activity/grade` | POST | LLM-grade a free-text (teach-back) answer, award XP on a pass. |
| `/activities` | GET | Public list of activity kinds. |
| `/flowy/quiz` | POST | Spontaneous quiz about a snippet (e.g. a paste). |
| `/flowy/quiz/answer` | POST | Award XP for a Flowy quiz answer. |
| `/settings` | GET | Read global settings (without revealing the API key). |
| `/settings/save` | POST | Persist model / base URL / API key. |
| `/state/wipe` | POST | Reset global XP (`scope: global`) or clear one notebook's checkpoints. |

All progression mutations follow the same shape — they take the current state, apply a pure idempotent change in `gamification.py`, and return the new state. The canonical XP total is owned by the server (`progress_store.py`); only `difficulty` + generated `quizzes` are persisted client-side in `metadata.flowquest` (see `questStore.ts`).
